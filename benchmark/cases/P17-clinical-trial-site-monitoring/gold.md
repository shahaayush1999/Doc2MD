## Page 09 - Source hierarchy and correction handling

| Priority | Source class | Controls when | Does not erase |
| --- | --- | --- | --- |
| 1 | Signed source correction / signed DCF | Identifies field, date, author, reason, and final entry | Original visible entry |
| 2 | Signed contemporaneous source form | No later signed correction exists for that field | Separate systems and later events |
| 3 | Verified external final report | Report identity and amendment state are established | Earlier report version |
| 4 | System audit trail | Timestamped state is reported as of its snapshot | Later signed source |
| 5 | Monitoring note | Describes review, lineage, or unresolved evidence | Underlying clinical source |
| 6 | Draft email or working tracker | Reported only as draft context | Any signed or verified record |

1. Report the original and corrected values when both are visible; label which value controls and preserve authorship and time.
2. A checkbox state carries meaning only with its exact label. Checked, unchecked, crossed, and disabled are distinct states.
3. A query-console snapshot remains valid evidence of its own timestamp even when a later signed clarification changes the field value.
4. A laboratory amendment supersedes the analytical value it identifies; custody timestamps remain independent facts.
5. An action due date in a signed CAPA controls over a working-tracker date, while the draft date remains part of the audit trail.
6. When no source controls a field, retain the unresolved state instead of choosing the most plausible value.
